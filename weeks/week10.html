Week 10 summary

  1. POST, PUT, and DELETE Request

    2. In this week's assignment, we learned about how to make POST, PUT, and DELETE requests in JavaScript using postman

  2. HTTP Methods
    • HTTP GET means you are requesting a resource
    • HTTP POST means you are sending a new resource for insertion
    • HTTP PUT means you are sending updates to a resource
    • HTTP DELETE means you are requesting a resource be deleted
  3. HTTP Request

    4. HTTP Request are strings that specify additional information

    HTTP Request contain 4 pieces of information

    • The endpoint (or URL)
    • The HTTP method
    • A series of HTTP headers
    • The data (or message body)
  4. Postman

    5. Postman is an API tool that makes it easy to interact with REST API

  5. Making POST, PUT, and DELETE using fetch API
  6. Sending a POST request

    7. A post request is is used to specify that data be added to the data store

    For HTTP POST request, you typically specify:

    • "POST" for the method
    • A string of JSON or form data for the body
    • A content-type HTTP header specifying the content-type
  7. Sending JSON DATA in a POST Request
  8. Sending PUT Request

    9. A PUT request is used to specify that a resource in the data store be edited

    For an HTTP PUT Request you typically specify:

    • "PUT" for the method
    • A string of JSON or form-data for the body
    • an HTTP header that specifies the content-type
